# <a name="obtain-details-using-azure-application-consistent-snapshot-tool-preview"></a>Obtención de detalles con la herramienta Azure Application Consistent Snapshot (versión preliminar)

En este artículo se ofrece una guía para ejecutar el comando de detalles de la herramienta Azure Application Consistent Snapshot que puede usar con Azure NetApp Files.

## <a name="introduction"></a>Introducción

El comando `azacsnap -c details` enumera los detalles de las instantáneas (nombre del volumen, nombre de la instantánea, hora de creación, comentarios y tamaño de la instantánea) almacenadas en todos los volúmenes del archivo de configuración.  La salida de la consola se puede pegar en una hoja de cálculo para su posterior análisis.

Debido a la información almacenada en los sistemas de back-end, la salida del comando es ligeramente diferente cuando se ejecuta en **Azure (instancias grandes)** o en **Azure NetApp Files**.

## <a name="command-options"></a>Opciones de comando

El comando `-c details` tiene las siguientes opciones:

- `--details snapshots` Proporciona una lista de detalles básicos acerca de las instantáneas de cada volumen que se ha configurado. Este comando se puede ejecutar en el servidor principal o en un servidor en la ubicación de recuperación ante desastres. El comando proporciona la siguiente información desglosada por cada volumen que contiene instantáneas:
  - Tamaño de las instantáneas totales de un volumen.
  - Cada instantánea de ese volumen incluye los detalles siguientes:
  - Nombre de la instantánea
  - Hora de creación
  - Tamaño de la instantánea
  - Frecuencia de la instantánea

- `--details replication` Proporciona los detalles básicos del estado de replicación desde el sitio de producción al sitio de recuperación ante desastres. El comando proporciona la salida para permitir la supervisión a fin de garantizar que se está llevando a cabo la replicación.  Muestra el tamaño de los elementos que se están replicando. También ofrece una guía de lo que se debe hacer si la replicación tarda demasiado o si el vínculo no está activo.

- `[--configfile <config filename>]` es un parámetro opcional que permite nombres de archivo de configuración personalizados.

### <a name="output-of-the-azacsnap--c-details---details-replication-command"></a>Salida del comando `azacsnap -c details --details replication`

Este comando comprueba el estado de replicación de almacenamiento desde el sitio principal a la ubicación de recuperación ante desastres y *debe* ejecutarse en el **servidor del sitio de recuperación ante desastres**. Este comando comprueba **solo** los volúmenes del archivo de configuración.

> [!NOTE]
> Esta opción solo está disponible en sistemas de **Azure (instancias grandes)** (HLI).

```bash
azacsnap -c details --details replication
```
